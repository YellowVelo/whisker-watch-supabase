# Feature Documentation Triage — /docs/review-features

**Date:** 2026-07-15
**Method:** Read every file in `docs/review-features`, cross-checked claims (UI screens, data fields, scoring/status logic, user flows) against the current codebase — primarily `src/lib/checkin/{scoring,config,chipLabels,checkinClient}.js`, `src/api/entities.js`, `supabase/migrations/*.sql`, `supabase/functions/*`, and the relevant pages/components.

**Key finding that drives most classifications below:** the app has gone through **two full scoring-system replacements**. `src/lib/checkin/scoring.js` states this explicitly: it "replaces every prior scoring system (Wellness Score V1's `computeDayScore`/`computeTrend`/`scoreLabel`/`explainScore`, Health Score V2's `computeHealthScore`/`computeHealthScoreDirection`)." The current model (migration `0026_vibe_and_symptom_count.sql`, "spec v5") uses a subjective **Vibe** (`daily_check_ins.status`: `great` / `off` / `tough` / `skipped`) plus an objective, unweighted **Symptom Count** — no 0–100 score, no 0–10 Health Score, no "Stable/Declining/Monitor" labels anywhere in the current UI. Any doc describing those is deprecated for that content, even if the surrounding feature (Home, Pets, Trends) is still very much alive.
